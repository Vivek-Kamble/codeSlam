Code Slam

About Code Slam

Code slam is an annual event held at Sai Krupa Enterprises, a franchise of Aptech Computer Education. The purpose of this event is to inspire students to write code in a fun way with others. Students will learn how to working in a team, delegate tasks to different members and finally deliver a product that is useable as per industry standards.

At the end of the competition, experts who work in the field will evaluate the project and give helpful suggestions to students.

Code Slam 2019

This year's Code Slam will focus on building an entire website from scratch, focussing on three areas of web development.

  1. Front End: Use HTML/CSS to design a fron end for a website.
  2. Back End: Use any server side language viz. ASP.Net, Node.JS, PHP or any other language to get data from the front-end and send it to a webservice
  3. Code Integration: Ensuring that the Front-End and Back end communicate and work with each other seamlessly.

Code Slam 2019 -- TASK

Overview

To create a website for a company that is the business of software development and training. The company is currently trying to develop a new Machine learning algorithm on movie tagging.

For example a new movie named "Toy Story" is released with following description

A cowboy doll is profoundly threatened and jealous when a new spaceman figure supplants him as top toy in a boy's room
The tags for this movie are
Animation, Adventure, Comedy

The above data will be added to a database already containing a huge number of such entries. This has already been done for you

If suppose a new movie called "Interstellar" is released with the following description

A team of explorers travel through a wormhole in space in an attempt to ensure humanity's survival.

Your job is to use machine learning to predict the tags for this movie, from the description itself

The Design

The website should contain the following pages

  1. A Home Page - divided in the following sections
    1. An "introduction" section that contains information about the company.
    2. A "gallery" section showcasing the premises and office space of the company
    3. A "Meet our staff" section containing information about some of the employees of the company. Viz - Name, designation, experience etc
    4. A projects section that shows information about notable projects handled by the company
    5. A "Contact us" section that contains address of the company shown on a map. A contact form to ask any query to the Customer Service department of the company
  2. Another page to see the list of movies present on the server
    1. A "Get Movies" button to get the list of all movies on the server
    2. The movies should be displayed on the same page
    3. A drop down list containing two options
      1. Random Forest
      2. Decision Tree
    4. A Create model button, which when clicked will create a Machine learning model in the background using the ML Model selected in the drop down list
  3. A form page where users will enter the movie name and description - the page should contain the following
    1. A form to enter the data, viz name, description etc
    2. A button to submit the data
    3. A section of the page itself where the tags calculated by the Machine Learning Algorithm will be displayed
    4. Remember to design the page well using CSS

Task for Students who have learnt Node.JS, PHP, ASP.Net or Java

We have already created a service which will be used to retrieve movies from the server

Your job will be to ensure the following.

  1. The messages from the contact form are added to a database. The user is then automatically sent an email message stating that "we have received your query and we will revert to you within 48 hrs"
  2. To handle the request from the "movie" page and forward it to the appropriate server that will apply the machine learning algorithm to calculate the tags. You don't have to create the service. It will be created for you

Task for Students who have learnt ML

When the create model button is pressed, an ML model must be created.

Your job is to predict the tags, using an ML model

Further instructions are given in a seperate PDF

User Task Flow

  1. User visits the home page
  2. User will try to send a test message to the site owner. Using the contact form on the home page
  3. User will then move to the create model page
  4. User clicks the "Get Movies" button
  5. This will retreive a list of movies
  6. User then clicks the "Create Model" button
  7. User then moves to the "Predict Tags" form page
  8. User enters the movie name and description
  9. Use clicks on the predict button
  10. The computer will predict the tags based on the model so created.
  11. The tags will be displayed on the screen

Marking Guidelines

HTML / CSS Marks

Code Quality - 20 Marks
Sr. No Category Description Marks
1 Code Indentation Proper use of tab spaces, brackets in HTML / CSS files 5
2 Basic SEO Proper use of tags like meta, strong, alt attribute among others 5
3 Reuse / proper naming of classes in CSS Classes in CSS should be reused to keep code repitition at minimum. 10

User Interface - 30 Marks
1 Position / Spacing of Interactive Elements How and where have you used your links, inputs, buttons or any other areas, in which user interaction is expected. The interface should be easy to understand and natural. 10
2 Validation Basic HTML validation when needed must be there in all forms to prevent common mistakes 10
3 Other UI elements Any other UI elements like navigation bar, banner etc and their relative position on the page. 10

User Experience - 50 Marks
1 Look and feel User experience is a subjective concept and varies from person to person. The general guideline is to ensure that your site "looks and feels good". This part is where all the pages of your site will judged and an overall marks will be given 15
2 Fluency in design The look and feel of all pages must be similar. The user should "feel" that he/she is still on the same site. As an example: if the main page has a "red" theme, the subpages cannot have a "blue" theme 10
3 Responsiveness in Design Your website should work well regardless of the device used to view it. Design for mobile, tablets and desktop resolutions 25
Back End Code JS/Angular/Server Side - Marks

Code Quality - 100 marks
Sr. No Category Description Marks
1 Code Indentation / Readability Proper use of tab spaces, brackets in your code files. Is the code easier to read? 10
2 Basic Documentation Proper use of comments in code. Tip: Don't comment too much, just as much as you think is needed. 10
